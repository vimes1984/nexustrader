## Session 7: Trade Paralysis Breakthrough — Bot Now LIVE TRADING (2026-07-21 22:00-22:15 UTC)

### Critical Bug Discovery
After 38+ minutes of the bot running with zero trades despite active signals, discovered **4 compounding root causes**:

### Root Cause 1: Dynamic min_win_rate Death Spiral 🌀
**File**: `probability_engine.py:243-257`
**The bug**: When win rate fell below 30%, `dyn_min` was RAISED to 0.63 (from 0.55 baseline). All p_win signals scored 0.50-0.56 — below the dynamically raised threshold. No trade could ever pass viability check.
**The fix**: LOWER `dyn_min` when losing (escape death spiral): `dyn_min = max(self.min_win_rate - 0.10, 0.40)` for WR < 0.3. Now takes MORE shots when behind, fewer when ahead.

### Root Cause 2: Silent TypeError in get_equity()
**File**: `execution_engine.py:382`
**The bug**: `self.get_equity()` called without required `current_prices` arg → TypeError caught silently by outer try/except → all open_position calls crashed without traceback.
**The fix**: `self.get_equity(getattr(self, 'last_known_prices', {}))`

### Root Cause 3: Kelly Fraction Floor Too Small
**File**: `evaluation/position_sizing.py:107`
**The bug**: `min_safe_fraction = 0.005` → $90.82 * 0.005 = $0.45 < $10 minimum position size → ALL trades rejected.
**The fix**: Raised floor to `0.06` (→ $5.45), lowered min position to $5.

### Root Cause 4: Cold-Start Safe Fraction
**File**: `evaluation/position_sizing.py:95`
**The bug**: Initial `safe_fraction = 0.02` → $90.82 * 0.02 = $1.82 < $5 minimum.
**The fix**: Raised to `0.05`.
