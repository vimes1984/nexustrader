How the Policy Network Fits In

The 8D state vector (RSI, MACD, BB, ATR, OU regime, OU theta, sentiment, 10-trade win rate) is fed into a small MLP. It outputs 12 strategy weights — one per strategy in the ensemble. After each trade closes, the realized PnL backpropagates through the network (REINFORCE + replay buffer).

The network does not predict prices. It learns which strategies to trust in the current market regime. This is layered on top of OU regime detection and performance biasing.

                    How the Bot Trades
                

                
                

                    

                        
Signal Generation

                        

                            12 individual strategies each produce a signal per tick (-1=short, 0=neutral, +1=long).
                            Their outputs are combined into one weighted signal.
                        

                        

                            Strategies: EMA crossover, RSI reversal, Bollinger reversion, Kalman filter,
                            stop hunt detector, random forest, momentum, volatility regime, mean reversion, trend follow, scalping, breakout
                        

                    


                    

                        
Weighting System

                        

                            Three layers determine which strategies matter most right now:
                        

                        
    
                            
  1. OU Regime Detection — estimates Ornstein-Uhlenbeck mean-reversion speed.
                            Trending markets favor trend/breakout/momentum. Mean-reverting markets favor RSI/BB/kalman.
    2. 
                            
  2. Performance Biasing — each strategy tracks its rolling win rate (last 20 trades).
                            Above 60% gets boosted. Below 35% gets suppressed.
    3. 
                            
  3. Policy Network — an 8x12x12x12 MLP takes the market state vector
                            (RSI, MACD, BB distance, ATR ratio, OU theta, OU regime, sentiment, win trend)
                            and outputs a softmax weight distribution. Trained via REINFORCE on trade PnL.
    4. 
                        

                        

                            Code: strategy_engine.py (ensemble + performance), learning_engine.py (policy network), quant_utils.py (OU estimation)
                        

                    


                    

                        
Entry / Exit Rules

                        

                            A trade triggers when the weighted signal exceeds thresholds:
                            >= +0.35 for long, <= -0.35 for short.
                            Once in a trade, stop-loss and take-profit are set per tick.
                            SL = entry +/- ATR * sl_multiplier. TP = entry +/- ATR * tp_multiplier.
                            Default multipliers: SL=1.5, TP=3.0 (configurable per ticker).
                            Cooldown period after a losing trade prevents revenge trading.
                        

                        

                            Code: main.py process_tick(), probability_engine.py
                        

                    


                    

                        
Position Sizing

                        

                            Three caps applied in order. The tightest one wins:
                        

                        
    
                            
  1. Calibration cap — Brier score of recent probability predictions.
                            Well-calibrated (Brier <0.20) => 5% of cash max. Poor (Brier >0.25) => 2% max.
    2. 
                            
  2. Drawdown penalty — linear taper. At 0% drawdown => full cap. At 50%+ drawdown => 0% of cap.
    3. 
                            
  3. Kelly fraction — estimated from actual closed-trade win/loss distribution. Usually 0.1–0.25.
    4. 
                        

                        

                            Formula: evaluation/position_sizing.py. Risk mode is "aggressive" or "conservative" (halves all sizes).
                        

                    


                    

                        
Safety System

                        

                            KillSwitch — monitors daily PnL, position count, and total exposure.
                            Trips and blocks all trading if any limit is exceeded. State persists across restarts.

                            DrawdownTracker — tracks peak-to-trough equity. Adjusts risk as drawdown grows.

                            MutationFreeze — agent-suggested parameter changes log for review but are not applied automatically.
                            Human approval required via the Optimizations Manager tab.
                        

                        

                            Code: evaluation/safety.py. Status endpoint: /api/safety/status.

                        Architecture — What Actually Runs
                    

                    

                        What follows is the real architecture. No imaginary daemons or fake cron agents.
                        These are the actual source files, their runtime responsibilities, and their data flows.
                    

                


                
                

                    

                        PER-TICK RUNTIME FLOW (process_tick, called ~1-10s per ticker in live mode)
                    

                    
data_ingestion.py → Fetch price from CCXT (Kraken). Compute RSI, MACD, BB, ATR. Tokenize into 32-token vocab.

                    
└─ row = {"close", "rsi", "macd", "bb_upper", "bb_lower", "atr", ...}


                    
main.py (process_tick) →

                    
├─ quant_utils.estimate_ou_process() → OU theta + regime (trending/mr)

                    
├─ learning_engine.get_state_vector() → 8D vector (MLP) or 24-seq token embeddings (LSTM/Transformer)

                    
├─ learning_engine.select_weights(state) → PolicyNetwork (MLP/LSTM/Transformer) → 6 weights

                    
├─ strategy_engine.get_weighted_signal() → weighted signal (-1..1)

                    
├─ 12 strategies : momentum_breakout, mean_reversion, trend_following, breakout_scalp, rsi_divergence, bb_squeeze (6 strategies per ticker after 72→36→6 purge)

                    
├─ probability_engine.evaluate_trade() → p_win, EV, Kelly fraction, is_viable

                    
├─ evaluation.safety.KillSwitch.check() → blocks if drawdown/position/exposure limits exceeded

                    
├─ execution_engine.open_position() → CCXT order (live) or simulated fill (paper/sim)

                    
├─ execution_engine.update_positions() → check SL/TP every tick

                    
└─ broadcast_message() → WebSocket push to dashboard


                    
ON TRADE CLOSE (on_trade_closed callback)

                    
├─ evaluation.safety.KillSwitch.record_trade(pnl)

                    
├─ evaluation.safety.DrawdownTracker.update(equity)

                    
├─ strategy_engine.record_trade_outcome() → update rolling win rates

                    
└─ learning_engine.learn_from_trade() → PolicyNetwork backprop

                


                
                    
LLaMA INFERENCE PATH (async, per tick)

                    
├─ llm_client.analyze_sentiment() → llama.cpp server (chris-System:8080)

                    
├─ llm_client.classify_regime() → market regime detection

                    
└─ llm_client.explain_trade() → trade rationale generation
                


                
                

                    

                        data_ingestion.py
                        
Fetches historical/candles from CCXT (Kraken). Computes RSI, MACD, Bollinger Bands, ATR. Runs live WebSocket streams via ccxt.pro. Optional CSV reload for sim.

                    

                    

                        strategy_engine.py
                        
12 strategy classes with per-strategy signals. Ensemble weights from PolicyNetwork. OU regime detection. Performance biasing (rolling win rate).

                    

                    

                        learning_engine.py
                        
Policy network — 3 architectures: MLP (8→12→6), LSTM (TokenEmbedder→64d→LSTM→Softmax), Transformer (4-head MHA→MeanPool→Softmax). Pure NumPy. REINFORCE-like policy gradient with replay buffer. Brain persistence + weight migration in DB. 73 unit tests.

                    

                    

                        execution_engine.py
                        
Position management (open/close/update). Live order routing via CCXT. SL/TP tracking. Cooldown system. Balance persistence to DB.

                    

                    

                        probability_engine.py
                        
ATR-based SL/TP. Win prob from signal + RSI + historical similarity. Kelly sizing. Drawdown-aware + calibration-capped position sizing.

                    

                    

                        quant_utils.py
                        
OU process estimation (OLS on log prices), regime detection, signal normalization utilities.

                    

                    

                        evaluation/ package
                        
safety.py (KillSwitch, DrawdownTracker, MutationFreeze), metrics.py (Sharpe, Sortino, Kelly), cost_model.py, harness.py, position_sizing.py, singletons.py

                    

                    

                        database.py
                        
SQLite WAL. Tables: settings, trades, portfolio_history, policy_brains, active_assets, notifications, backups, agent_runs, optimizations.

                    

                    

                        main.py
                        
FastAPI server. Orchestrator class. Tick processing loop. WebSocket broadcast. REST API + dashboard serving. Startup/shutdown hooks.

                        Neural Network Architectures (3 modes, configurable via nn_architecture)
                    

                    

                        

                            MLP (Default)

                            8 features → 12 hidden ReLU → 6 strategy Softmax. Backward compatible, always available.

                            learning_engine.py / PolicyNetwork
                        

                        

                            LSTM

                            TokenEmbedder (32 vocab → 64d) → 2-layer LSTM → Softmax. Sequence-aware.

                            tokenizer.py + sequential_policy_net.py
                        

                        

                            Transformer

                            4-head MHA encoder (2 layers) → Mean Pool → Softmax. Attention Is All You Need.

                            multi_head_attention.py + transformer_policy_net.py
                        

                    

                    

                        All pure NumPy, no dependencies. Full JSON serialization for DB persistence. 73 unit tests pass across all NN modules.
                        Toggle via nn_architecture DB setting (requires restart). Manage in the LLM tab.
